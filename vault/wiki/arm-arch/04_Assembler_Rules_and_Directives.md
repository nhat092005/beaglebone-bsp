# Chapter 4: Assembler Rules and Directives

## 4.1 Structure of Assembly Language Modules

The general form of source lines in assembly files is:
`{label} {instruction|directive|pseudo-instruction} {;comment}`

- **Labels**: Represent an address in memory. Must start at the beginning of the line (column 1). Can only be defined once.
- **Instructions/Directives**: Must be preceded by whitespace (tab or spaces).
- **Comments**: Keil uses a semicolon (`;`). CCS allows an asterisk (`*`) or semicolon (`;`) in column 1, or a semicolon anywhere else.

**Unified Assembler Language (UAL)**: The current format for writing code that can be assembled for ARM, Thumb, or Thumb-2.
- *Old format*: `MOV <Rd>, <Rn>, LSL shift`
- *UAL format*: `LSL <Rd>, <Rn>, shift`

## 4.2 Predefined Register Names

The assembler recognizes case-sensitive predefined names for registers:
- `r0-r15` or `R0-R15`
- `s0-s31` or `S0-S31` (floating-point)
- `a1-a4` (synonyms for r0-r3, used for arguments/results)
- `sp` or `SP` (Stack Pointer, r13)
- `lr` or `LR` (Link Register, r14)
- `pc` or `PC` (Program Counter, r15)
- `cpsr` / `CPSR` (Current Program Status Register)
- `spsr` / `SPSR` (Saved Program Status Register)
- `apsr` / `APSR` (Application Program Status Register)

## 4.3 Frequently Used Directives

### Defining a Block of Data or Code

**Keil Tools**: `AREA sectionname{,attr}{,attr}…`
- Must have at least one `AREA` directive.
- Attributes: `CODE` (default `READONLY`), `DATA` (default `READWRITE`), `ALIGN = expr` (aligns on 2^expr byte boundary).

**Code Composer Studio (CCS)**: `.sect "sectionname"`, `.text` (default code section), `.data`, `.bss` (uninitialized).

### Register Name Definition
Assigning a custom name to a register for readability.
- **Keil**: `name RN expr` (e.g., `coeff1 RN 8`)
- **CCS**: `.asg "character string", substitution symbol` (e.g., `.asg R13, STACKPTR`)

### Equating a Symbol to a Numeric Constant
Similar to `#define` in C. Does not place anything in memory.
- **Keil**: `name EQU expr{,type}` (e.g., `SRAM_BASE EQU 0x04000000`)
- **CCS**: `symbol .set value` or `symbol .equ value`

### Declaring an Entry Point
Tells the linker where execution begins. Only one per program.
- **Keil**: `ENTRY`

### Allocating Memory and Specifying Contents
Places data directly into memory at assembly time.

**Keil Tools**:
- `DCB`: Bytes (8-bit). Example: `C_string DCB "C_string", 0` (null-terminated string).
- `DCW` / `DCWU`: Halfwords (16-bit). Aligned / Unaligned.
- `DCD` / `DCDU`: Words (32-bit). Aligned / Unaligned.

**CCS Tools**:
- `.byte`, `.char`: 8-bit. Example: `LAB1 .byte 10, -1, "abc", 'a'`
- `.half`, `.short`: 16-bit.
- `.word`, `.int`, `.long`: 32-bit.
- `.float`, `.double`: Floating-point.

### Aligning Data or Code
Pads memory with zeros to ensure the next address starts on a specific boundary.
- **Keil**: `ALIGN {expr{,offset}}`. If `expr` is omitted, defaults to a word (4-byte) boundary.
- **CCS**: `.align {size in bytes}`. Default is 1. Size of 4 aligns to a word boundary.

### Reserving a Block of Memory
Reserves zero-initialized space.
- **Keil**: `{label} SPACE expr` (expr = number of bytes).
- **CCS**: `{label} .space size_in_bytes` (label points to first byte) or `{label} .bes size_in_bytes` (label points to last byte).

### Assigning Literal Pool Origins
Literal pools hold constants generated by the assembler (e.g., from `LDR` pseudo-instructions). By default, they are placed at the `END` directive. If code is too large (>4KB offset limit), place `LTORG` manually.
- **Keil**: `LTORG`. Must be placed where execution cannot reach it (e.g., after an unconditional branch or return).

### Ending a Source File
- **Keil**: `END`
- **CCS**: `.end`

## 4.4 Macros

Macros allow text substitution at assembly time to replace repetitive code blocks. (Unlike subroutines, they increase code size but avoid call overhead).

**Keil Syntax**:
```asm
MACRO
{$label} macroname{$cond} {$parameter{,$parameter}…}
; code using parameters (e.g., $vara)
MEND
```

## 4.5 Assembler Operators

Primitive operations performed *by the assembler* on data/constants before they are used in an instruction.

| Operation | Keil | CCS |
| :--- | :--- | :--- |
| Modulo | `A:MOD:B` | `A%B` |
| Shift Left | `A:SHL:B` | `A<<B` |
| Shift Right | `A:SHR:B` | `A>>B` |
| Rotate Left | `A:ROL:B` | |
| Rotate Right | `A:ROR:B` | |
| Bitwise AND | `A:AND:B` | `A&B` |
| Bitwise OR | `A:OR:B` | `A|B` |
| Bitwise XOR | `A:EOR:B` | `A^B` |

**Example Usage**:
Setting specific bits in a constant to load into a register:
```asm
MOV    r0, #((1:SHL:14):OR:(1:SHL:12))  ; Keil
MOV    r0, #((1<<14) | (1<<12))         ; CCS
```

## 4.6 Math Functions in CCS

CCS provides built-in math functions for generating compile-time data (e.g., lookup tables).
- `$$cos(expr)`, `$$sin(expr)`, `$$log(expr)`, `$$max(e1, e2)`, `$$sqrt(expr)`

Example:
```asm
.float  $$cos(0.434)
```
